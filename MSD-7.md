# MSD-7 (Microservice Design): Собирайте отзывы о ваших проектах обслуживания

Дата: 23-12-2023

## Статус (Status)
Принято

## Контекст (Context)
Начальная версия API и сервиса, спроектированная на OAS, но этого не достаточно.
Нужно показать экскизный проект конечных точек разработчикам клиетнов.

## Решение (Decision)
Получить отзывы о проектах сервисов

## Следствие (Consequences)
Проектирование сервиса не заверщается, пока его проект не будет представлен целевой аудитории, а пполученные отзывы не будут учтены.

При разработке сервисов и API необходимо иметь в виду две группы клиентов:
* конечные пользователи системы - гарантирует правильность создания системы
* разработчики клиентов, которые будут программировать в/д с вашими сервисами - гарантирует удобство ее использования
